## 核心议题

CHAOSS（Community Health Analytics for Open Source Software）是一个由 Linux 基金会支持的项目，专门研究和标准化开源社区健康指标。Matt 和 Sean 介绍了他们如何把这套经过十年验证的指标体系引入 Web3 公共物品评估领域。

---

## 主要内容

**CHAOSS 的起源**
CHAOSS 项目始于2017年，由 Linux 基金会、Mozilla 和 GitHub 等合作推动。目标：建立一套标准化的开源项目健康指标，让资助方、贡献者和用户都能客观评估一个开源项目是否"健康"。

**什么是"社区健康"**
CHAOSS 定义的社区健康覆盖多个维度：
- **贡献活跃度**：提交频率、活跃贡献者数量
- **多样性**：贡献者的地理、性别、组织多样性
- **可持续性**：核心贡献者分散度（不依赖单一贡献者）
- **响应性**：Issues 和 PR 的处理速度
- **包容性**：新贡献者是否容易上手

**Web3 的特殊挑战**
把 CHAOSS 指标应用到 Web3 公共物品时遇到的新挑战：
- 链上活动数据分散在多条链上
- 匿名性使得跨项目的贡献者追踪困难
- 代码贡献以外的社区贡献（治理参与、社区支持）更难量化

**CHAOSS 与 Open Source Observer 的合作**
CHAOSS 和 Open Source Observer（OSO）正在合作，把 CHAOSS 的标准化指标应用到链上数据分析中，为 RetroPGF 等资助评估提供更可靠的数据依据。

**贡献者多样性指标**
CHAOSS 特别关注社区的多样性指标：
- 如果一个项目的核心贡献者90%来自同一国家/组织，这是风险信号
- 性别多样性虽然难以从 GitHub 数据推断，但 CHAOSS 提供了调查工具
- 新贡献者留存率反映了社区的包容性

**指标不是目标，是信号**
Matt 强调：指标只是健康状态的信号，不应该成为目标本身（Goodhart's Law："当一个指标成为目标，它就不再是好指标"）。团队需要理解指标背后的逻辑，而不只是追求数字好看。

---

## 关键观点

> "开源社区健康不是一个数字，而是一个多维的状态——就像人类健康一样，需要用多种指标综合判断，而不是靠单一指标下结论。"

- 十年的开源社区数据积累让 CHAOSS 的指标体系比大多数 Web3 原生方案更成熟
- 多样性和包容性指标是最难量化但最重要的社区健康维度

## 墨尔本本地 LGBTQ+ 社群的启发

**多样性指标对 LGBTQ+ 社群内部同样重要**：LGBTQ+ 社群自身也可能产生内部多样性不足的问题——领导层是否代表了跨性别者、残障酷儿、有色人种酷儿？用类似 CHAOSS 的多样性指标定期检视社群参与结构，是防止少数群体边缘化的主动工具。

**健康指标追踪社群活力**：可以借鉴 CHAOSS 的框架为社群设计健康仪表板：新成员加入率、活跃成员留存率、活动参与多样性、决策参与度——定期回顾这些指标，及早发现社群活力下降的信号。

**Goodhart's Law 对社群的警告**：如果把"活动人数"作为唯一评估指标，社群可能开始追求人数而非质量；如果把"代币持有量"作为影响力指标，有钱的成员就会主导社群。始终保持对"指标 vs 真实价值"的区分意识，是健康社群文化的关键。
